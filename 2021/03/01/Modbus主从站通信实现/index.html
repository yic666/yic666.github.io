Modbus主从站通信实现

一、Modbus Poll & Modbus Slave

1.介绍

Modbus Poll ：Modbus主机仿真器，用于测试和调试Modbus从设备。该软件支持ModbusRTU、ASCII、TCP/IP。用来帮助开发人员测试Modbus从设备，或者其它Modbus协议的测试和仿真。它支持多文档接口，即，可以同时监视多个从设备/数据域。每个窗口简单地设定从设备ID，功能，地址，大小和轮询间隔。你可以从任意一个窗口读写寄存器和线圈。如果你想改变一个单独的寄存器，简单地双击这个值即可。或者你可以改变多个寄存器/线圈值。提供数据的多种格式方式，比如浮点、双精度、长整型（可以字节序列交换）。

Modbus Slave： Modbus从设备仿真器，可以仿真32个从设备/地址域。每个接口都提供了对EXCEL报表的OLE自动化支持。主要用来模拟Modbus从站设备,接收主站的命令包,回送数据包。帮助Modbus通讯设备开发人员进行Modbus通讯协议的模拟和测试，用于模拟、测试、调试Modbus通讯设备。可以32个窗口中模拟多达32个Modbus子设备。与Modbus Poll的用户界面相同，支持功能01, 02, 03, 04, 05, 06, 15, 16, 22和23，监视串口数据。

（3）读线圈状态

在Modbus Slave 中双击任何一个线圈的值，会弹出如下的对话框，可以选择线圈的开启还是关闭，此时将第0个线圈开启

可以在Modbus Poll中看到主机的数据也会随之改变

打开串口收发数据帧监视信息对话框，可以看到数据帧发生了改变，从

000241-Tx:01 01 00 00 00 0A BC 0D
000242-Rx:01 01 02 00 00 B9 FC

变成了

000243-Tx:01 01 00 00 00 0A BC 0D
000244-Rx:01 01 02 01 00 B8 6C

也就是数据变成从00变成了01，这里用的是16进制来表示数据

报文分析如下表

从机地址 功能码 线圈起始地址 读取线圈个数 CRC校检
01 01 00 00 00 0A BC 0D
从机地址 功能码 返回字节个数 线圈数据1 线圈数据2 CRC校检
01 01 02 01 00 B8 6C

（4）写线圈状态

在Modbus Poll 中点击05，使用05功能码实现对线圈状态的写入，也可以直接双击对应的线圈值进行修改

修改之后，在Modbus Slave中可以看到对应的值发生了改变

串口收发数据帧监视信息对话框，可以看到数据帧为

000523-Tx:01 05 00 00 FF 00 8C 3A
000524-Rx:01 05 00 00 FF 00 8C 3A

报文分析

从机地址 功能码 设置地址 设置内容 CRC校检
01 05 00 00 FF 00 8C 3A

也可以通过功能码15同时写多个线圈

数据帧如下：

000697-Tx:01 0F 00 00 00 0A 02 3B 00 F6 08
000698-Rx:01 0F 00 00 00 0A D5 CC

报文分析

从机地址 功能码 线圈起始地址 设置长度 发送字节个数 设置内容 CRC校检
01 0F 00 00 00 0A 02 3B 00 F6 08

（5）补充

上述实现了功能码01，05，15对线圈状态的读写，功能码02与01的操作类似，至于03，06，16这三个功能码也与上诉的操作差不多，只是操作方式为字操作。

4.串口调试助手的使用

下面以报文

000697-Tx:01 0F 00 00 00 0A 02 3B 00 F6 08
000698-Rx:01 0F 00 00 00 0A D5 CC

为例，使用串口调试助手，手工发出 MODBUS 主、从站该发出的数据，并观察对方发出的数据。

打开两个串口调试助手，按下图设置，可以模拟之前VSPD设立的虚拟串口

在COM1的发送栏输入01 0F 00 00 00 0A 02 3B 00 F6 08 然后点击发送，在COM2的接收区域就能看到收到的报文

在COM2的发送栏输入01 0F 00 00 00 0A D5 CC就能返回报文

二、modbus_tk库的使用

1.介绍

modbus tk是使用python语言实现的modbus协议栈， 该函数库可以实现TCP以及RTU传输方式。

下载地址：https://github.com/ljean/modbus-tk

2.主机

下载对应pip

pip install modbus_tk

示例代码

import serial

import modbus_tk
import modbus_tk.defines as cst
from modbus_tk import modbus_rtu

PORT = "COM1"
# 此处可修改使用的串口

def main():
        master = modbus_rtu.RtuMaster(
            serial.Serial(port=PORT, baudrate=9600, bytesize=8, parity='O', stopbits=1, xonxoff=0)
        )
        # 设置设备的端口号、波特率、通信数据长度、奇偶校验、停止位、流控制
        master.set_timeout(5.0)
        master.set_verbose(True)
        read = []
        read = master.execute(1, cst.READ_COILS, 0, 10)
        #master.execute(从机ID，功能码，读取位置，数据长度)

        print(read)
        return read

if __name__ == "__main__":
    main()

3.从机

示例代码

import modbus_tk
import modbus_tk.defines as cst
from modbus_tk import modbus_rtu
import serial

PORT = "COM2"
# 此处可修改使用的串口

def main():
    """main"""
        #Create the server
    server = modbus_rtu.RtuServer(serial.Serial(PORT))

    try:
        server.start()

        slave = server.add_slave(1)
        #添加地址为1的从机
        slave.add_block('0', cst.HOLDING_REGISTERS, 0, 10)
        #添加取名为0的0~10内存块，设置为保持寄存器（03功能码）
        slave.set_values('0', 0, [1, 1, 0, 1, 1, 0, 1, 1])
        #设置寄存器数值
        while True:
                values = slave.get_values('0', 0, 10)
                #读取寄存器数值0~10
                print('values read: %s\r\n' % (str(values)))                            

    finally:
        server.stop()

if __name__ == "__main__":
    main()

运行结果如下

values read: (1, 1, 0, 1, 1, 0, 1, 1, 0, 0)

在Modbus Poll 可以看到主从站通信正常（Err是因为之前还没运行代码）
